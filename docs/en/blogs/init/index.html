Init: One-Shot AGENTS.md for the Project

First step on a new repo: write the rules down, stop re-explaining them.

💡

In short

/init is not a new protocol: it packages “scan repo → write rules” as a repeatable prompt so every project can start with solid agent conventions.

Background

Without AGENTS.md, agents guess scripts and package boundaries. Stale or scattered CLAUDE/Cursor rules raise migration cost; ad-hoc “write me an AGENTS” drifts in quality.

@piex-dev/init: install once, then /init creates or improves project-root rules in place.

How it works

Pi only loads AGENTS.md/CLAUDE.md; builtins never generate them. A package pi.prompts entry turns prompts/init.md into /init: expand to a full user message, let the model investigate and write.

Value is the investigation checklist: prefer executable sources, keep high-signal facts, improve existing files instead of blind rewrites.

Implementation

Prompt-only package: no extension, no peer deps. After write, remind /reload so the session picks up context. No global ~/.pi/agent/AGENTS.md writes, no interactive menu.

Design notes

OpenCode ships built-in /init + initialize.txt: we keep the checklist, writing rules, $ARGUMENTS, and in-place improve; we drop kernel embedding and opencode-specific config refs, shipping a pi prompt package instead.

oh-my-pi focuses on multi-format context loading, not generation: leave loading to pi’s walk-up; piex fills the “write the rules” gap.

Roadmap

Optional thin extension for path injection and exists-hints; post-write checklist; /init global; merge with session rule-learning later.
