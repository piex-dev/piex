Context: Token Usage at a Glance

Who's eating context, and which turns are blowing up — one command.

💡

In short

/context shows exactly what's inside your session: who's consuming the context window, which entries are bloated, and where your token budget is going.

Background

pi's built-in /session only lists sessions — no per-session content analysis. @piex-dev/context fills this gap with structured entry analysis, role breakdowns, and token estimates.

How it works

Token estimation via chars/3.5 (conservative for mixed content but useful). Three analysis axes: Assistant (model output, usually dominant), User (instructions), Tool Results (easiest to bloat). Output as ASCII bar chart + Markdown table.

Implementation

~160-line single file. Handles both string and content-block-array formats. Counts by role/type. Complements pi /session: /session for list, /context for distribution.

Roadmap

Precise token counting (once tokenizer API available); time-series growth chart; anomaly detection for oversized tool results; configurable estimation coefficients.
