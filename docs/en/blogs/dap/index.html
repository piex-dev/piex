DAP: Real Debugging for Coding Agents

Stop print-debugging. Launch, stop, and inspect.

💡

In short

With @piex-dev/dap, agents can run and inspect instead of only guessing. Not a test suite—but variables become readable facts.

Background

Guess–edit–rerun is brittle for runtime bugs. Humans use debuggers; agents shouldn’t be stuck with print-debugging.

@piex-dev/dap exposes a debug tool over the Debug Adapter Protocol: launch, breakpoints, stepping, variables.

How it works

Agent intent → DAP client → adapters (debugpy, dlv, lldb, …). Session states running/stopped/terminated over stdio JSON-RPC. Fourteen adapters selected via file type and root markers. TCP remote attach is intentionally out of scope for now.

Implementation

Framed client, session manager with breakpoint queues, output caps, idle cleanup, capability checks, and non-interactive env defaults.

Roadmap

Richer breakpoints, stop-default context packs, project launch configs, optional TCP, and eval metrics for real debug success.
