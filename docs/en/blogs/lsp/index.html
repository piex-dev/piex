LSP: Language-Server Eyes for Agents

Not chattier agents—agents that can check the truth.

💡

In short

@piex-dev/lsp doesn’t make agents chat more; it lets them ask IDE questions: errors, definitions, references.

Background

Without LSP, agents lean on read/grep and guesswork. Signature changes miss call sites; type errors wait for full builds.

@piex-dev/lsp adds diagnostics, go-to-def, references, hover, symbols, and format.

How it works

Match servers by filetype, find roots via markers, speak JSON-RPC over stdio, cache processes. Diagnostics come from push notifications.

Implementation

Self-contained client + large defaults.json. Nine actions; subset of omp; pairs with hashline and dap in a static→edit→dynamic loop.

Roadmap

Rename/codeAction first, indexing readiness, graded diagnostic digests, project overrides, optional post-edit checks.
