2026-07-19 LSP Diagnostics Extension

LSP: Language-Server Eyes for Agents

The win is automatic ERRORs after edit—not hoping the model remembers to ask.

💡

In short

@piex-dev/lsp answers IDE questions—and appends ERRORs after edit/write so the model does not have to remember to re-check.

Background

Without LSP, agents lean on read/grep and guesswork. Signature changes miss call sites; type errors wait for full builds.

@piex-dev/lsp adds navigation, diagnostics, rename/code actions, and a post-edit ERROR loop.

How it works

stdio JSON-RPC; rootMarkers for discovery; publishDiagnostics cache. On edit/write: sync disk (didOpen/didChange) → wait → ERROR-only cap attached to the tool result.

Implementation

initOptions/settings wired correctly; multi-server diagnostics with linter vs primary routing; rename defaults to preview; WorkspaceEdit confined to cwd. Disable post-edit with PI_LSP_DIAGNOSTICS_ON_EDIT=0; override any server's command with PI_<NAME>_LSP_COMMAND.

Diagnostics correctness: push diagnostics settle after a quiet period (800ms default, per-server diagnosticsSettleMs) so slow servers that publish empty-then-real can't report a dirty file as clean; servers advertising diagnosticProvider get LSP 3.17 pull diagnostics instead. codeAction/resolve is only called when resolveProvider is advertised; overlapping TextEdits are refused; server stderr is captured into error messages; Windows .bat/.cmd spawn via cmd.exe.

Version history

The 0.2.0 lesson: servers like intelephense push a batch of empty diagnostics first, then the real ones — returning immediately misreports a faulty file as clean; on crash only an exit code remained, so debugging was guesswork. 0.3.0 fills in the protocol details — settle quiet-period waits for push to stabilize, pull diagnostics let the server compute on demand, stderr lands in the error message, resolveProvider gating avoids redundant requests to unsupported servers, and overlapping-TextEdit detection prevents file corruption. These are protocol lessons pi-extensions pi-lsp ran first; piex merges them into the persistent-process architecture, paying cold-start only once.

Roadmap

Done: correctness, post-edit diagnostics, rename/code_actions, diagnostics settle/pull, overlapping-edit guard. Tests: bun test extensions/lsp/test. Next: project .lsp.json, module split, indexing readiness, directory-wide batch diagnostics. Deferred: lspmux, auto-download, completion.
