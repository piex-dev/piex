BTW: Ephemeral Side Questions

Side questions with full context. Answers leave no trace.

💡

In short

/btw lets you ask a quick side question with full session context — quietly, like leaning over to a colleague. The answer won't affect future conversation.

Background

The classic dilemma: you want to ask something related to the task but don't want the agent to treat it as task context. Opening another session is slow; asking inline pollutes context. @piex-dev/btw's answer: the question never reaches the agent — the extension takes a snapshot of the conversation and calls the model out-of-band, leaving nothing in the session.

How it works

A direct completeSimple call carries a ≤40K-char conversation snapshot (user/assistant + tool-call summaries, truncated from the head), and the answer renders in an abortable, scrollable Markdown pager. Question and answer never enter the session — pollution-freedom is architectural, not a runtime filter.

~/.pi/piex-dev/btw/btw.json can pin a dedicated model (provider/model-id) and thinkingLevel; credentials resolve via modelRegistry.getApiKeyAndHeaders, falling back to the current model with a warning — cheap model for side questions, strong model for the main conversation.

Implementation

~700-line single file. Core modules: loadCompleteSimple (pi-ai 0.79/0.80 compat loading), resolveBtwModel (configured → current model fallback chain), buildConversationContext (sessionManager.getBranch snapshot), BtwAnswerPager (j/k, PgUp/PgDn, Home/End, progress %). Keeps the interactive ctx.ui.input prompt when invoked without arguments.

Version history

VersionArchitectureNotes
0.1.0Inject + filter[BTW] question injected into the main agent loop; tools banned via prompt; context hook filtered the pair afterwards using a "skip the next assistant" heuristic
0.2.0 (current)Out-of-band callOne-shot completeSimple request with a ≤40K conversation snapshot; nothing enters the session. Adds btw.json model/thinkingLevel settings, abortable loader, Markdown pager

The 0.1.0 lesson: "pollute first, clean up later" is hard to get right in an asynchronous message stream — toolCall sequences broke the filter heuristic, queued turns misaligned the marker, and the injected instruction message itself became another artifact to clean. 0.2.0 simply doesn't pollute, and the whole class of edge cases disappears.

Roadmap

Relevance-aware snapshots (e.g. always keep the last toolResult in full); multi-turn follow-ups inside the pager (still session-free); a /btw model quick-config command.
