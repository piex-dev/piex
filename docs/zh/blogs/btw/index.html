BTW：不被记住的临时提问

临时提问上下文，答案不留痕。

💡

导语

/btw 让你用满上下文问一个临时问题，答案不进入后续对话——就像会议里压低声音问旁边人一句。

问题背景

coding agent 的经典困境：想问跟任务有关但又不想它「当真」的问题。另开 session 太麻烦，直接问又污染上下文。@piex-dev/btw 的回答是：问题压根不发给 agent——扩展自己拿一份会话快照旁路调模型，session 里什么都不会留下。

技术原理

扩展绕开 agent loop，用 pi-ai 的 completeSimple 旁路调用：构建 ≤40K 字符的会话快照（user/assistant + tool call 摘要，头部截断），一次性请求模型，答案渲染进可中断、可滚动的 Markdown pager。问答从不写入 session，防污染是架构保证，而非运行时过滤。

~/.pi/piex-dev/btw/btw.json 可配独立 model（provider/model-id）与 thinkingLevel，凭据经 modelRegistry.getApiKeyAndHeaders 解析，失败回退当前模型并告警——临时提问可以用便宜模型，主对话继续用强模型。

实现方案

约 700 行单文件。核心模块：loadCompleteSimple（pi-ai 0.79/0.80 兼容加载）、resolveBtwModel（设置模型→当前模型回退链）、buildConversationContext（sessionManager.getBranch 快照）、BtwAnswerPager（j/k、PgUp/PgDn、Home/End、进度百分比）。保留无参数时 ctx.ui.input 交互输入。

设计参考

项目机制piex 取舍
oh-my-pi /btw引擎级旁路提问采纳「不写入 session」的目标语义
pi-extensions pi-btwcompleteSimple 旁路 + 会话快照 + settings 文件 + loader/pager UI采纳整体架构，保留 piex 的无参交互输入；pi-ai 0.79/0.80 兼容加载直接复用

版本迭代

版本架构说明
0.1.0注入 + 过滤[BTW] 问题注入主 agent loop，靠 prompt 禁工具，事后用 context 钩子按「跳过下一条 assistant」推断过滤
0.2.0（当前）旁路调用completeSimple 一次性请求 + ≤40K 会话快照，问答不写入 session；新增 btw.json 独立模型/thinking level、可中断 loader、Markdown pager

0.1.0 的教训：「先污染、后治理」在异步消息流里很难做严——toolCall 序列让过滤推断错位漏滤，agent 忙时排队让 btw 标记盖错 turn，注入的指令消息本身也成了要清扫的制品。0.2.0 换成「不产生污染」，整类边界随之消失。

优化计划

快照可按问题相关性选择性携带（如始终保留最后一个 toolResult 全文）；pager 内支持多轮追问（仍不写入 session）；补 /btw model 快捷配置命令。
