2026-07-19 Init AGENTS.md Prompt

Init：一键写出项目的 AGENTS.md

新项目第一步：把规则写进仓库，而不是每次口头重讲。

💡

导语

/init 不发明新协议：把「扫仓库 → 写规则」收成一条可重复的 prompt，让每个新项目第一步就能建好 agent 约定。

问题背景

没有 AGENTS.md，agent 只能猜 scripts 与包边界；规则过时或散落在 CLAUDE/Cursor 文件里，迁移成本高。质量靠口头「帮我写个 AGENTS」会漂移。

@piex-dev/init：pi install 一次，之后 /init 即可创建或就地改进项目根规则。

技术原理

pi 只加载 AGENTS.md/CLAUDE.md，内置 slash 没有生成命令。package 的 pi.prompts 把 prompts/init.md 变成 /init：展开为完整 user message，由当前模型调研并写文件。

价值在调查清单：可执行源优先，只留高信号事实，已存在则 in-place 改进。

实现方案

纯 prompt 包，无 extension、无 peer 依赖。写完提示 /reload 使当前会话生效。不碰全局 ~/.pi/agent/AGENTS.md，不做交互菜单。

设计参考

OpenCode 内置 /init + initialize.txt：采纳调查清单、写作规则、$ARGUMENTS、就地改进；不采纳内核内置与 opencode 专用配置，改为 pi prompt 包。

oh-my-pi 强化多格式 context 加载、无生成侧：加载层交给 pi 原生 walk-up，piex 只补「写出规则」。

优化计划

可选 thin extension 注入路径与存在性提示；生成后 checklist；/init global；与会话规则学习衔接 merge。
