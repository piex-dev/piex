Review：把 Code Review 做成 Agent 的一等公民

可重复的评审能力，而不是随手贴 diff。

💡

导语

/review 把「贴一段 diff 让 AI 看看」收成可重复能力：范围清楚、噪声可控、人机同一套引擎。

问题背景

「让 AI 看看代码」卡在：diff 从哪来、lock/min/dist 噪声、提示词漂移、人机两套入口。

@piex-dev/review：人用 /review，模型调 review 工具，同一套 git diff + 噪声过滤 + 结构化 prompt。

技术原理

模式：uncommitted / staged / vs base / commit / custom。解析 unified diff，剔除 lock、构建产物、vendor、媒体等；过大 diff 不内嵌全文，提示按需 read。

实现方案

约 350 行单文件。交互 select 菜单后 sendUserMessage followUp。是 omp review 轻量版：无多 agent 并行与 TUI overlay。

设计参考

项目机制piex 取舍
oh-my-pi review/review 命令 + review 工具；多 agent 并行评审 + TUI overlay；diff 引擎 + 噪声过滤采纳：diff 引擎、噪声过滤、多模式、人机共用 buildReviewPrompt。不采纳：多 agent（依赖 subagent）、TUI overlay（改 sendUserMessage followUp）
OpenCode /review内置 /review 命令，review 未提交或指定 commit/branch借鉴：交互菜单模式选择、统一 prompt 模板、多 source 评审

核心取舍：完整保留 diff 引擎与噪声过滤，放弃并行与视觉 overlay。等 subagent 成熟再迁多 agent 版。

优化计划

findings JSON；与 lsp/dap 联动；项目级 ignore；多 pass；远期 subagent 版。
