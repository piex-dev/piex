2026-07-19 LSP Diagnostics Extension

LSP：给 Agent 一双「语言服务器」的眼睛

不是更会聊天，而是能问有没有错、定义在哪。

💡

导语

@piex-dev/lsp 不是让 agent 更会聊天，而是让它在改代码前后能问到和 IDE 同一类问题：有没有错、定义在哪、引用还有谁。

问题背景

没有 LSP 时，agent 只靠 read/grep 与脑补类型。改签名漏调用点、红线要等整次构建才知道。人编辑器交给语言服务器；agent 也该能问。

@piex-dev/lsp 提供 diagnostics / definition / references / hover / symbols / format。

技术原理

按后缀匹配 server，root markers 定 workspace，stdio JSON-RPC，会话内缓存进程。诊断来自 publishDiagnostics 推送缓存，不是轮询。

实现方案

单文件 lsp.ts + 数据驱动 defaults.json（大量 server 默认）。九个 action 覆盖读智能与格式化；无 TUI 高亮，是 omp 的子集。

推荐闭环：lsp 弄清改哪 → hashline 安全改 → 再 diagnostics → 运行时问题上 dap。

优化计划

优先 rename/codeAction；indexing 就绪状态；诊断分级摘要；项目级 overrides；edit 后可选自动诊断。
