Context：一眼看清 Token 用量

谁在吃上下文、哪个 turn 膨胀最快，一个命令看清。

💡

导语

/context 一眼看清当前 session 的内容分布：谁在占用上下文、哪些条目膨胀最快、你的 token 预算都花在哪了。

问题背景

pi 内置的 /session 只能列 session 列表，缺单 session 的内容分析。@piex-dev/context 填补这个缺口：结构化条目分析、角色占比和 token 估算。

技术原理

用 chars/3.5 估算 token（中英混合偏保守但够用），三个分析轴：Assistant（模型输出，通常占大头）、User（指令）、Tool Results（最易意外膨胀）。输出 ASCII bar chart + Markdown 表格。

实现方案

约 160 行单文件。适配 string 和 content block 数组两种格式，按 role/type 分类统计。与 pi /session 互补：/session 看列表，/context 看分布。

优化计划

精确 token 计数（等 tokenizer 接口）；时序增长图（定位哪个 turn 膨胀最快）；异常检测（自动标记过大的 tool results）；可配置度量系数。
